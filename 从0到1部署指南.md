## 1. 灵感起源

### Karpathy 的"一篇笔记哲学"

2025 年，AI 研究员 Andrej Karpathy 写了一篇博客叫 [*The append-and-review note*](https://karpathy.bearblog.dev/the-append-and-review-note/)，分享了他坚持多年的笔记方法：

- **只维护一个文本文件**，用苹果备忘录
- 任何想法、待办、灵感，**追加到文件顶部**
- 不分类、不打标签、不建文件夹
- 偶尔翻一翻，已经被反复忽略的内容自然"沉到底部"
- 真正重要的东西会被反复"救"到顶部

他的核心洞察是：**分类和整理是认知负担**。一个好的笔记系统不应该要求你成为图书管理员。

### AI 时代的新可能

Karpathy 的"append-and-review"本质上是一个**手动执行的生物记忆模型**——新信息进来，旧的往下沉，被反复回顾的停留，不再相关的遗忘。

但如果加一层 AI 呢？

如果不需要手动翻找，而是 AI 每天晚上自动读取你一整天的对话，帮你蒸馏出核心洞察？如果每天早上打开终端，AI 就能告诉你过去几天你在思考什么？如果你聊到外部知识，AI 自动帮你研究、归档、提取概念？如果每 2-4 周，AI 自动从你的思考中识别出反复出现的底层思维框架，固化为"锚点"？

这就是 My_LLM_Wiki 的起点。

## 2. 核心理念

### 知识库不是仓库，是会新陈代谢的生命体

传统知识库在 5-10 年尺度上遭遇两大灾难：

1. **信息熵爆表**：海量旧数据污染检索空间，信噪比崩溃
2. **知识库硬化**：旧思想无法修正，失去自我迭代能力

My_LLM_Wiki 的解法是**模拟人类记忆的巩固机制**——不是把所有东西塞进硬盘，而是让信息经历一条从短期到长期的"代谢链"：

```
聊天对话 ──→ Logs ──→ Digest ──→ Wiki_Grid ──→ soul_profile ──→ Archives
(秒-分)     (日)     (日-周)     (月-年)        (周-月)          (年+)

短时记忆 → 工作记忆 → 长时记忆 → 灵魂锚点 → 历史冰封
```

### 四层地质架构

```
00_Raw_Materials/    原始素材层    【只读】   外部世界流入的定型内容
10_Wiki_Grid/        知识晶格层    【读写】   一生沉淀的结构化晶体智识
20_Templates/        模板契约层    【只读】   所有产出物的格式约束
30_Daily_Stream/     时间流层      【追加】   时间轴上的流动记忆
```

每层有严格的权限边界（只读/追加/精准修改），数据永远按一个方向流动，绝不回写。

## 4. 部署 4 步走

### Step 1：克隆仓库

```bash
git clone [你的仓库地址]
cd My_LLM_Wiki
```

### Step 2：用 Obsidian 打开

```
启动 Obsidian → Open folder as vault → 选择 My_LLM_Wiki 文件夹
```

Obsidian 提供双链图谱可视化和 Markdown 编辑体验。**不打开 Obsidian 也能用**，系统在终端中完整运行。

### Step 3：启动 OpenCode

```bash
opencode
```

OpenCode 启动后会自动加载 `opencode.md`（系统宪法）和 `soul_profile.md`（精神画像）作为 System Prompt，确保 AI 理解你的知识管理规则。

### Step 4：开始首次对话

新会话启动后，系统自动执行两件事：
- **morning-sync**：扫描近 5 天日刊，生成思想快照（首次使用会提示"无近期日刊"）
- **reminder_check**：检查 6 项周期任务，有需要时输出提醒

此时输入 `digest` 或任何问题，系统即进入正常运作。

### 每层的权限与设计意图

#### 00_Raw_Materials/ — 原始素材层（只读）

> **"外部世界流入的定型内容"**

这一层的东西是已经写定的——一本书的章节、一场演讲的文字稿、一篇新闻报道。放进来后**只追加不修改**。如果需要更正，写入新文件并标注版本，不覆盖旧内容。

**设计意图**：源材料是知识的"原材料"。原材料不可篡改——就像你不会去修改一本已经出版的书。

**归类决策**：
```
这段内容来自哪里？
├── 书籍章节/全文    → Books/
├── 演讲/论文/访谈   → Papers/
└── 网页/博客/新闻   → WebClips/
```

#### 10_Wiki_Grid/ — 知识晶格层（精准读写）

> **"一生沉淀的结构化晶体智识"**

这是系统的核心资产。从对话框提炼出来的结构化知识单元，经过验证、经过关联、成网状排列。

| 子目录 | 归类条件 | 模板 |
|--------|----------|------|
| `00_Anchors/` | 底层人生观、核心决策框架、反复出现的思维范式 | 自由格式 |
| `Concepts/` | 原子化的核心概念（Zettelkasten 风格），一张卡一个概念 | `Template_Concept.md` |
| `Entities/` | 人、公司、地点、产品、事物 | `Template_Entity.md` |
| `Syntheses/` | 跨领域关联洞察，不属于任何单一概念 | 自由格式 |

**修改策略**：不全文重写卡片。新观点与旧卡片冲突时，不覆盖，而是追加辩证区块（`> [!danger] 认知演进与范式转移`），保留认知演化轨迹。

#### 20_Templates/ — 模板契约层（只读引用）

> **"所有产出物的格式约束"**

概念卡片、实体卡片、日刊都有严格的前端元数据（Frontmatter）规范。不符合模板的产出将被拒绝写入。

**设计意图**：模板是系统的"语法"。所有卡片遵循统一结构，保证跨时间、跨主题的可对比性和可检索性。修改模板后，所有新产出自动生效。

> **注意**：日刊模板（`Template_Digest.md`）的原始章节名与 `digest/SKILL.md` 中 AI 实际执行的蒸馏规范存在差异。SKILL.md 规定了三模块蒸馏的具体要求（核心灵感捕获 + 知识深度拓展 + 涟漪映射建议），AI 执行蒸馏时以 SKILL.md 为准。如需统一，修改模板即可全局生效。

#### 30_Daily_Stream/ — 时间流层（追加/生成）

> **"你和 AI 对话的时间轴"**

| 路径 | 用途 | 写入方式 |
|------|------|----------|
| `{year}/{MM_Mon}/Logs/` | 每天原始对话，一问一答格式 | 每轮对话自动追加 |
| `{year}/{MM_Mon}/Digests/` | 每晚蒸馏的思想日刊 | 输入 `digest` 生成 |
| `Archives/` | 往年已完成的年份数据冰封 | 年度结晶时自动迁移 |

**设计意图**：时间是知识的天然维度。昨天聊过什么、上周在思考什么、上个月在纠结什么——按时间轴组织，检索时有明确的"热/温/冷"温度分层。

#### .opencode/ — 系统控制层

> **"系统的中枢神经"**

```
.opencode/
├── config.json          ← 模型参数、技能开关、检索权重、提醒间隔
├── paths.json           ← 全系统唯一路径配置中心
├── config_schema.json   ← 配置结构合法性校验
└── skills/              ← 17 个 Python 脚本 + 4 个 SKILL.md
```

**关键设计**：所有脚本从 `paths.json` 读取路径，不在代码中硬编码。修改目录名只需改一处。`config.json` 中所有参数集中管理。

## 6. 单向数据流与知识反哺循环

### 数据永远单向流动

```
Logs ──→ Digests ──→ Wiki_Grid ──→ soul_profile.md ──→ Archives
```

- **绝不反向写入**：Digest 不会回写 Logs，Wiki_Grid 卡片不会回写 Digest
- **绝不循环引用**：soul_profile 引用 Wiki_Grid 的概念，但 Wiki_Grid 不会引用 soul_profile

### 完整代谢链路

```
你每说一句话
  │
  ├─→ [log-append] 静默追加到 Logs/YYYY-MM-DD_Chat.md
  │
  ▼ 一天结束后，你输入 digest
  │
  ├─→ [digest] 读取今日 Chat Log
  │      ├─ 模块1：核心灵感捕获（1-3条洞察）
  │      ├─ 模块2：知识深度拓展（增值分析）
  │      └─ 模块3：涟漪映射建议（关联已有知识卡片）
  │  写入 Digests/YYYY-MM-DD_Digest.md
  │
  ▼ 次日早晨
  │
  ├─→ [morning-sync] 扫描近5天 Digest → 300字思想快照 → AI知道你的上下文
  │
  ▼ 每2-4周
  │
  ├─→ [anchor] 扫描4周 Digest → 识别底层思维框架 → 写入 00_Anchors/
  │
  ▼ 每周
  │
  ├─→ [soul_updater] 重写 soul_profile.md（顶部更新/底部存历史）
  │
  ▼ 每年
  │
  └─→ [yearly_crystallize] 全年 Digest → 精华沉淀 Wiki_Grid + 冰封 Archives
```

### 三层检索路由（70/20/10 权重）

当你向 AI 提问时，系统按以下权重检索知识：

```
热数据（70%）：soul_profile.md + 近两周 Digests
   ↑ 感知你当前的精神状态和近期关注焦点

温数据（20%）：10_Wiki_Grid/ 全量
   ↑ 调取你一生沉淀的结构化常识

冷数据（10%）：30_Daily_Stream/Archives/
   ↑ 仅在明确提及历史话题时激活
```

## 7. 自动衍生机制详解

### 日刊蒸馏（digest）

输入 `digest`，AI 执行三模块蒸馏：

| 模块 | 做什么 | 质量标准 |
|------|--------|----------|
| **模块1：核心灵感捕获** | 从对话中提取 1-3 条深层洞察 | 独立可复用，6个月后阅读仍能理解 |
| **模块2：知识深度拓展** | 对讨论的核心框架做增值解析 | 增加对话中没有的学术/工程内容 |
| **模块3：涟漪映射建议** | 分析洞察与已有知识卡片的关联 | 用 `[[双链]]` 给出具体修改建议 |

生成结果按 `Template_Digest.md` 规范写入 `Digests/YYYY-MM-DD_Digest.md`。

**变体指令**：
- `digest YYYY-MM-DD` — 补生成指定日期
- `digest week` — 补生成本周全部缺失日刊
- `digest month` — 补生成本月全部缺失日刊

### 晨间唤醒 + 完整性核查（morning-sync）

每天首次对话自动触发，分三个阶段：

**Phase 1 — 思想快照**：扫描近 5 天日刊，压缩为 ≤300 字思想快照：
> 关注焦点 + 卡点/障碍 + 活跃思维线程 + 情绪趋势

**Phase 2 — 完整性核查 + 自动补齐**：运行 `reminder_check.py --catch-up`，检测缺失的日刊、逾期未执行的周/月任务，并自动补齐——自动生成缺失日刊、自动更新 soul_profile、自动执行锚点结晶。所有可自愈操作静默完成，仅汇报摘要。

**Phase 3 — 统一呈现**：将 Phase 1 的思想快照 + Phase 2 的补齐摘要 + Phase 2 的告警，合并为简洁的启动消息。

这意味着每天早上打开终端，AI 已经知道你这几天在思考什么，系统也已经自动修好了昨天该做但没做的事。

### 锚点结晶（anchor）

输入 `anchor`。每 2-4 周执行一次。AI 扫描近 4 周所有日刊，识别满足 2 条以上标准的候选框架：

1. 在 3 周以上的 Digest 中反复出现
2. 跨越至少 2 个不同领域/主题
3. 是"如何思考"的框架，而非"思考的答案"
4. 是你明确表达过"这是一种底层框架/思维模型"的观点

识别出的框架固化为锚点卡片写入 `00_Anchors/`。已有锚点只在新证据出现时追加"本周更新"区，不修改框架正文。

### 周报生成（weekly）

输入 `weekly`（周一自动执行）。扫描上周全部日刊，生成**评论刊物风格周报**——不是流水账汇总，而是：

> 本周图景 + 深度聚焦 + 边缘地带 + 数据断面 + 下阶段风向

写入 `04_Weekly_Report/{year}/{year}-W{week}_Weekly.md`。

### 灵魂更新（soul_updater）

每周由 AI 触发。读取本周所有日刊，渐进重写 `soul_profile.md` 顶部——更新你的当前精神状态、正在构建的思维模型、近期焦虑卡点。旧状态追加到文件底部的"历史版本"区。超过 2 年的历史版本迁移至 `soul_profile_archive/`。

### 外部知识摄入（ingest）

当你问到外部知识（"乔布斯2009年的演讲是什么？""帮我分析《思考，快与慢》"），AI 自动执行五步分发：

```
研究源材料
  └─→ 路由写入 Raw_Materials/{Books|Papers|WebClips}/
       └─→ 提取概念卡片 → Concepts/
            └─→ 提取实体卡片 → Entities/
                 └─→ 跨领域合成报告 → Syntheses/
```

也可以用显式指令：`ingest <查询>`（完整分发）或 `ingest raw <查询>`（仅归档源材料）。

### 自愈与维护

系统包含多个"后台进程"式的自愈机制：

| 机制 | 触发 | 功能 |
|------|------|------|
| **健康自检** | 会话启动/手动 `healthcheck` | 6项自检，能自愈的无感修复 |
| **链接修复** | 手动 `fixlinks` | 扫描破损双链，自动创建最小占位卡片 |
| **周期提醒** | 会话启动/手动 `remind` | 检查6项周期任务，输出待办清单 |
| **图谱熵控** | 定期手动 | 检测重复概念，合并重写双链 |
| **冲突消解** | 写入新概念时 | 新观点与旧卡片冲突 → 追加辩证区块 |

## 9. 使用节奏

### 典型一天

```
早上打开终端，启动 OpenCode
  ├─→ 自动 morning-sync：AI 知道你昨天在想什么
  └─→ 自动 reminder：提醒今天有没有该做的维护

白天自然对话
  └─→ 每句话自动记入 Logs/（静默，无感）

晚上结束时输入 digest
  └─→ AI 读今日对话 → 三模块蒸馏 → 写日刊
```

### 典型一周

```
周一：morning-sync 自动生成上周周报
每天：digest 生成日刊
周末：anchor 提取本周思维框架
随时：ingest 处理遇到的外部知识
```

### 典型一月

```
每 2-4 周：anchor 锚点结晶
每月：graph_entropy 扫描重复概念
月末：回顾本月 Digest → 发现认知演化轨迹
```

### 典型一年

```
春季：回顾 soul_profile 的渐进变化
年终：yearly_crystallize 全年知识结晶 → 精华入 Wiki_Grid → 冰封 Archives
跨年：soul_profile 超过 2 年的历史 → 迁移至 soul_profile_archive/
```

## 11. 常见问题

### Q: 没有 DeepSeek API Key 能用吗？

**完全可以。** 系统默认走"通道 B"蒸馏——AI 在会话窗直接读取对话记录并执行蒸馏，不需要任何外部 API。`digest auto` 是通道 A，仅在你有 API Key 且希望走脚本自动化时使用（通道 A 和 B 的蒸馏质量一致）。

### Q: 输入 digest 但今天没有对话记录？

系统会提示：
> 今天还没有对话记录。等我们聊完后再输入 digest 生成日刊。

### Q: digest week 跳过了某些日期？

两种情况：
- 该日期已有 Digest → 跳过（不覆盖已有日刊）
- 该日期没有 Chat Log → 跳过（没有对话可蒸馏）

### Q: 如何修改目录结构？

修改 `paths.json` 中的对应路径，所有脚本自动生效。不要直接在脚本中硬编码路径。

### Q: 旧对话日志占用太多空间？

不需要手动删除。它们被归档到 Archives 后，检索权重自动降至 10%，不影响系统性能。年度结晶后会自动冰封。

### Q: 如何添加新的自动技能？

1. 在 `.opencode/skills/` 下创建新的 Python 脚本或 `SKILL.md`
2. 在 `config.json` 的 `skills.enabled` 中注册
3. 如需新路径，在 `paths.json` 中添加映射

### Q: soul_profile.md 中能看到我的历史精神状态吗？

文件底部"历史版本"区保留了每周的精神画像快照。可以看到自己的认知演化轨迹——什么时候关注什么、焦虑什么、构建了什么新思维模型。
