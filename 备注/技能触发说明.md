## 一、OpenCode 原生技能（4 项，新会话自动加载）

这四项技能以 `SKILL.md` 格式存放在 `.opencode/skills/<name>/` 下，OpenCode 启动时自动加载，**无需手动配置或记忆**。

---

### 1. digest — 日刊蒸馏

| 项目 | 内容 |
|------|------|
| **触发方式** | 输入 `digest` |
| **变体** | `digest YYYY-MM-DD`（指定日期）、`digest week`（本周全量）、`digest month`（本月全量） |
| **输入** | `30_Daily_Stream/{year}/{month}/Logs/YYYY-MM-DD_Chat.md` |
| **输出** | `30_Daily_Stream/{year}/{month}/Digests/YYYY-MM-DD_Digest.md` |
| **工作流** | 读 Chat Log → 三模块蒸馏（核心灵感 + 深度拓展 + 涟漪映射）→ 按 Template_Digest.md 写入 |
| **是否需要 API Key** | 否（会话窗 AI 直接执行） |

**典型对话：**
```
你：digest
AI：Found 1 log file for 2026-05-27
    Distilling...
    Digest written: 30_Daily_Stream/2026/05_May/Digests/2026-05-27_Digest.md
```

---

### 2. ingest — 语料分析入库

| 项目 | 内容 |
|------|------|
| **触发方式** | 自然语言或显式指令 `ingest <查询>` |
| **变体** | `ingest raw <查询>`（仅归档源材料，不做分析提取） |
| **输入** | 用户对外部知识（演讲/书/论文/新闻）的查询 |
| **输出** | `00_Raw_Materials/{Books\|Papers\|WebClips}/` + `10_Wiki_Grid/Concepts/` + `Entities/` + `Syntheses/` |
| **工作流** | 研究源材料 → 路由写入 Raw_Materials → 提取概念 → 提取实体 → 跨领域合成 → 汇报清单 |
| **是否需要 API Key** | 否 |

**典型对话：**
```
你：乔布斯2009年的演讲是什么？
AI：（研究 → 写入 Raw_Materials/Papers/ → 提取概念/实体 → 汇报清单）
```

**内容类型路由规则：**

| 你说的是 | 归档到 |
|----------|--------|
| 演讲/Keynote/Talk | `Papers/` |
| 论文/学术期刊 | `Papers/` |
| 书/章节 | `Books/` |
| 网页/新闻/博客 | `WebClips/` |
| 访谈 | `Papers/` |

### 3. morning-sync — 晨间唤醒（含周一自动周报）

| 项目 | 内容 |
|------|------|
| **触发方式** | 每天首次对话**自动触发**；也可手动输入 `sync` |
| **输入** | 近 5 天所有 `_Digest.md` 文件 |
| **输出** | 300 字以内的思想快照（纯文本）→ 注入 System Prompt 顶部 |
| **工作流** | 扫描近5天 Digest → 压缩为 300 字快照（关注焦点 + 卡点 + 思维线程 + 情绪） |
| **是否需要 API Key** | 否 |

**效果：** AI 在每天早上第一次对话时，会先输出一段你的思想状态快照，让你确认它理解了你的上下文。相当于每天自动"交班"。**周一额外自动生成上周周报**，写入 `04_Weekly_Report/`。

---

### 4. log-append — 实时对话记录

| 项目 | 内容 |
|------|------|
| **触发方式** | **每轮对话自动触发（静默，不提示）** |
| **输入** | 用户消息 + AI 回复 |
| **输出** | 追加到 `30_Daily_Stream/{year}/{month}/Logs/YYYY-MM-DD_Chat.md` |
| **格式** | `## 问：HH:MM:SS` / `### 答：HH:MM:SS` |
| **是否需要 API Key** | 否 |

**效果：** 不需要你做任何事。你每说一句话、AI 每回一句，自动写入当天的 Chat Log。这是后续 `digest` 和 `morning-sync` 的数据基础。

## 二、关联技能（8 项，后台脚本/手动触发）

这些技能以 Python 脚本形式存放在 `.opencode/skills/` 下，通过对话窗中 AI 手动调用或定期触发。

| 脚本 | 中文 | 触发方式 | 用途 |
|------|------|----------|------|
| `eod_digest.py` | 日结晶脚本 | 对话窗 AI 自动调用（配合 digest 技能） | 提供 prepare/write 辅助函数 |
| `corpus_ingest.py` | 语料分析脚本 | 对话窗 AI 自动调用（配合 ingest 技能） | 路由/渲染/校验辅助函数 |
| `anchor_weekly.py` | 锚点生成脚本 | 对话窗 AI 自动调用（配合 anchor 指令） | Digest 扫描 + 锚点渲染 + 更新追加 |
| `reminder_check.py` | 周期提醒脚本 | 每次会话启动自动 / 手动 remind | 6 项任务时间戳检查 + 友好提醒 |
| `link_validator.py` | 链接修复脚本 | 手动 fixlinks | 破损链接扫描 + 自动模板化修复 |
| `weekly_report.py` | 周报生成脚本 | 对话窗 AI 调用（配合 weekly 指令） | Digest 扫描 + 数据收集 + 评论刊物风格渲染 |
| `morning_sync.py` | 晨间同步脚本 | 对话窗 AI 自动调用（配合 morning-sync 技能） | Digest 扫描与压缩 |
| `log_append.py` | 日志追加脚本 | AI 或你手动调用 | `python3 skills/log_append.py user "消息"` |
| `soul_updater.py` | 灵魂更新 | 每周由 AI 触发 | 读取本周 Digest → 重写 soul_profile.md |
| `yearly_crystallize.py` | 年度结晶 | 每年底由 AI 触发 | 全年 Digest → Wiki_Grid + 冰封 Archives |
| `graph_entropy.py` | 图谱熵控 | 定期手动触发 | 检测重复概念 → 合并重写双链 |
| `conflict_resolver.py` | 冲突消解 | 写入新概念时自动触发 | 旧卡片中追加辩证区块 |
| `ripple_weave.py` | 涟漪联动 | 修改卡片时自动触发 | 全库双链自动更新 |
| `ast_patcher.py` | 精准修改 | 按需调用 | Markdown AST 局部 patch |
| `hybrid_search.py` | 混合检索 | 按需调用 | 70/20/10 权重三层检索 |
| `memory_router.py` | 记忆路由 | 按需调用 | 热/温/冷数据分流 |

## 三、所有手动指令速查表

| 指令 | 行为 | 类型 |
|------|------|------|
| `digest` | 生成今天的思想日刊 | OpenCode 技能 |
| `digest YYYY-MM-DD` | 补生成指定日期日刊 | OpenCode 技能 |
| `digest week` | 补生成本周全量缺失日刊 | OpenCode 技能 |
| `digest month` | 补生成本月全量缺失日刊 | OpenCode 技能 |

## 四、完整数据流

```
你说话
  │
  ├─→ [log-append] 静默追加到 Logs/YYYY-MM-DD_Chat.md
  │
  ▼ 全天对话结束后，你输入 digest
  │
  ├─→ [digest] 读 Logs → 三模块蒸馏 → 写 Digests/YYYY-MM-DD_Digest.md
  │
  ▼ 次日早晨
  │
  ├─→ [morning-sync] 扫描近5天 Digest → 300字快照 → 注入 System Prompt
  ├─→ [reminder] 静默检查 6 项周期任务 → 有需要时输出提醒
  │
  ▼ 当你查到外部知识
  │
  ├─→ [ingest] 研究 → Raw_Materials → Concepts + Entities + Syntheses
  │
  ▼ 每 2-4 周
  │
  ├─→ [anchor] 扫描 Digest → 识别固化框架 → 写入 00_Anchors/
  │
  ▼ 每周/每年
  │
  ├─→ [soul_updater] 重写 soul_profile.md
  └─→ [yearly_crystallize] 年硬化 + 冰封 Archives
```
